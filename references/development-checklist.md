# 高保真开发自检清单

> 用途：在页面规格整理完成后、代码交付前，逐项检查页面是否真正按设计与规则还原，并尽可能逼近像素级表现。

## 1. 页面壳与整体层级

- [ ] 是否先明确了主布局规则与页面内部规则的边界
- [ ] 页面整体层级是否与设计一致
- [ ] 是否保留了设计中的主要容器层级
- [ ] 是否擅自减少了 DOM 层级
- [ ] 页面整体背景、滚动区域、固定区域是否一致
- [ ] Header / Sidebar / Content 的关系是否一致

## 2. 区块划分与模块结构

- [ ] 区块划分是否一致
- [ ] 卡片数量、顺序、嵌套关系是否一致
- [ ] 标题栏、内容区、底部操作区是否按设计拆分
- [ ] 右侧导航 / 指引区是否保留正确结构
- [ ] 分割线、间隔块、占位区是否保留
- [ ] 是否擅自合并了视觉模块

## 3. 尺寸与间距

- [ ] 宽高是否准确
- [ ] 最小宽度 / 最大宽度是否准确
- [ ] 内边距是否准确
- [ ] 外边距是否准确
- [ ] 行间距、列间距是否准确
- [ ] 标题与内容间距是否准确
- [ ] 按钮之间间距是否准确
- [ ] 固定底部操作栏的高度、padding、对齐是否准确

## 4. 字体与颜色

- [ ] 字体样式是否准确
- [ ] 标题字号、字重、行高是否准确
- [ ] 正文与辅助文字样式是否准确
- [ ] 颜色是否准确
- [ ] 文本、图标、背景、状态色是否准确

## 5. 边框、圆角、阴影

- [ ] 边框是否准确
- [ ] 边框颜色与粗细是否准确
- [ ] 圆角是否准确
- [ ] 阴影是否准确
- [ ] 渐变背景是否准确
- [ ] 是否额外增加了设计中不存在的阴影、描边或视觉特效

## 6. 表单布局与控件细节

- [ ] 表单布局是否一致
- [ ] `form-item` 是上下布局还是左右布局，是否与设计一致
- [ ] 表单项是双列还是单列，是否与设计一致
- [ ] 全局表单控件最小宽度规则是否遵守
- [ ] 组合输入框的下拉宽度与总宽度关系是否正确
- [ ] label / value / form-item 常见排布是否一致
- [ ] Arco Design 默认样式是否按项目规范完成覆盖
- [ ] hover / focus / error / disabled 等状态是否与规格一致

## 7. 交互与状态

- [ ] 显隐逻辑是否一致
- [ ] 展开 / 收起逻辑是否一致
- [ ] 新增 / 删除逻辑是否一致
- [ ] 联动逻辑是否一致
- [ ] 表单校验规则是否一致
- [ ] 默认值、回显、重置逻辑是否一致
- [ ] 操作栏是否一致
- [ ] 右侧指引、时间轴、步骤导航的激活逻辑是否一致
- [ ] Message / Toast / 全局提示在不同端上的行为是否一致

## 8. 浮层与特殊结构

- [ ] 弹窗方向与结构是否一致
- [ ] 抽屉方向与结构是否一致
- [ ] 弹窗 / 抽屉宽度是否按设备类型处理
- [ ] 时间轴、标签组、日期块、特殊卡片是否保留原始结构
- [ ] 渐变卡片、引导块、提示块是否未被普通卡片替代

## 9. 响应式与跨端

- [ ] 响应式是否一致
- [ ] PC 与移动端的模块显隐是否一致
- [ ] PC 与移动端的布局方向变化是否一致
- [ ] 双列改单列的断点与规则是否一致
- [ ] 右侧辅助区在移动端是否按规则隐藏
- [ ] 全局组件在 PC 与移动端是否按约定切换行为

## 10. 抽象边界与推断值

- [ ] 是否擅自抽象过度
- [ ] 是否为了复用破坏了还原度
- [ ] 是否用“工程优化”名义改动了布局方向或层级
- [ ] 是否存在未标记的推断值
- [ ] 所有推断值是否都明确标记为 `【推断值】`
- [ ] 所有高风险歧义是否都进入“待确认项”

## 11. 交付前复核

- [ ] 是否先输出了结构化 UI 设计说明书，再进入代码实现
- [ ] 是否先输出了开发方案，再进入代码实现
- [ ] 是否在最终说明中列出了风险点与待确认项
- [ ] 是否说明了哪些值来自设计、哪些值来自推断
- [ ] 是否保留了与设计一致的视觉层级、样式细节与交互逻辑

## 12. 布局意图与适配检查

- [ ] 是否区分了固定尺寸与布局推导结果
- [ ] 单子元素容器是否正确判断为铺满或内容自适应
- [ ] 双子元素容器是否正确判断谁铺满、谁自适应
- [ ] 三子元素容器是否按角色关系而非数量做布局
- [ ] 是否错误地将导航栏、标题栏、用户信息行做成三等分
- [ ] 主体内容区是否正确占据剩余空间
- [ ] 操作区是否按内容自适应
- [ ] 是否明确了允许滚动的容器
- [ ] 是否因为小数取整意外制造滚动
- [ ] 文本区域是否配置了合理的最大宽度或省略策略
- [ ] 适配后是否仍保持主要视觉关系不变

## 13. 多 node-id 结构解析检查

- [ ] 是否先完成了多个 node-id 的归类
- [ ] 是否识别了页面根节点
- [ ] 是否识别了一级模块和二级模块
- [ ] 是否明确了父子层级关系
- [ ] 是否明确了兄弟节点关系
- [ ] 是否识别了共同父容器
- [ ] 是否先确认了容器布局，再处理子节点样式
- [ ] 是否错误地把子节点当成页面一级模块
- [ ] 是否错误地把同一模块内部节点拆成多个独立区块
- [ ] 是否明确了滚动区 / 固定区
- [ ] 是否明确了浮层归属
- [ ] 是否在结构树未确认前就直接进入代码实现

## 14. 视觉归属与定位参考系检查

- [ ] 是否区分了视觉归属、DOM 挂载归属、定位参考归属
- [ ] 是否错误地把容器内 absolute 元素实现成 viewport 级 fixed 元素
- [ ] 是否明确了底部操作栏实际挂载在哪个父容器下
- [ ] 是否明确了定位参考容器
- [ ] 是否检查了宽度是否应跟随父容器而非整页铺满
- [ ] 是否检查了左右边界是否受父容器 padding 影响
- [ ] 是否检查了该元素是否覆盖父容器内容
- [ ] 是否为正文内容预留了必要的底部空间
- [ ] 是否错误地把视觉上的“底部栏”理解成全局 footer
- [ ] 是否错误地把视觉上的“悬浮块”理解成独立页面级模块

## 15. 文本与排版细节检查

- [ ] letter-spacing 是否准确
- [ ] 文本单行 / 多行 / 省略处理是否准确
- [ ] 文本容器高度与垂直居中方式是否准确
- [ ] 标签、按钮、输入框内文字是否视觉居中
- [ ] 数字、时间、状态文案是否存在视觉偏移
- [ ] 文本对齐方式是否与设计一致
- [ ] 行数限制与换行行为是否与设计一致
- [ ] 是否将 `font-family` 与字号 / 字重 / 行高 / 字距分开处理，而不是把字体名当作必须还原项

## 16. 定位与层级检查

- [ ] 是否保留了 Figma 中的精确坐标关系
- [ ] `absolute` / `fixed` / `sticky` 的参考容器是否正确
- [ ] z-index 层级是否与设计一致
- [ ] 遮罩层覆盖范围是否正确
- [ ] popup / dropdown / tooltip / dialog / drawer 的锚点关系是否正确
- [ ] 是否保留了浮层对齐方式，而不只是“弹出来”

## 17. 组件默认样式覆盖检查

- [ ] Arco 默认样式是否已完整识别
- [ ] Button 默认高度、padding、圆角是否与设计一致
- [ ] Tag 默认 line-height、padding 是否已校正
- [ ] Input / Select / DatePicker / Radio / Checkbox / Switch 等组件的默认样式是否已覆盖
- [ ] Modal / Drawer / Table / Tabs / Collapse 等结构组件默认样式是否已检查
- [ ] 是否存在沿用组件库默认视觉而偏离设计稿的情况

## 18. 状态与交付边界检查

- [ ] 是否明确了默认态
- [ ] 是否明确了首轮必须实现的状态
- [ ] 是否把仅作参考的状态错误纳入默认态
- [ ] 是否把局部状态稿误实现为独立页面
- [ ] 是否把浮层态误实现为页面固定模块
- [ ] 是否明确区分了默认态、首轮实现态、延后实现态和参考态

## 19. 信息来源与数值一致性检查

- [ ] 是否遵守了信息优先级规则
- [ ] 是否存在 Figma 原值被无说明地调整或统一化
- [ ] 是否存在人工修正规格被旧 Figma 值覆盖
- [ ] 是否所有偏离设计原值的实现都已显式说明
- [ ] 是否所有设计原值、推断值和实现偏差都被正确区分

## 20. 字体授权与字体实现检查

- [ ] 是否识别了 Figma 中的 `font-family` 信息
- [ ] 是否默认忽略了未经确认授权的字体名
- [ ] 是否没有把 Figma 字体名直接写死进代码
- [ ] 是否优先使用了项目已有字体方案或系统安全字体栈
- [ ] 是否通过字号、字重、行高、字距、对齐、容器高度等方式逼近视觉结果
- [ ] 是否存在为了追求还原度而引入未知商业字体的情况
- [ ] 是否在说明书中明确说明了 `font-family` 的处理策略
